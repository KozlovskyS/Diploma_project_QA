# Тестирование мобильного приложения "В хосписе"
---

## Описание 

Мобильное приложение "В хосписе" реализует следующий функционал:

информация о новостях;
функционал для управления и добавления новостей;
тематические цитаты;
информацию о приложении.

Автоматизированное тестирование включает в себя проверку основных функций (авторизация пользователя, переходы по страницам, просмотр, создание и редактирование новостей, просмотр тематических цитат и информации о приложении).
По результатам формируется отчет о проведенном автоматизированном тестировании с помощью **Allure**.

---

## Подготовка тестового окружения 

1. На рабочую станцию установить интегрированную среду разработки **Android Studio Narwhal 4 Feature Drop | 2025.1.4.**
2. На рабочую станцию установить систему формирования отчетов **Allure**
3. Склонировать [репозиторий](https://github.com/KozlovskyS/Diploma_project_QA) в локальную папку
4. Открыть в **Android Studio** папку **fmh-android** 
5. Синхронизировать проект
6. Создать и запустить эмулятор в **Android Studio** с характеристиками  Pixel 4 API 31 (Android 12, RAM 2 Gb) (анимацию окон и переходов отключить) 
---

## Порядок запуска автотестов

В Android Studio на папке проекта **fmh-android\app** выбрать пункт контекстного меню **Run 'All Tests'** либо нажать **Ctrl+Shift+F10**, либо в терминале ввести команду **./gradlew cAT**

---

## Создание отчета **Allure**

1. После прохождения  скопировать результаты тестирования с тестового устройства на рабочую станцию:
  * В **Android Studio** запустить **Device File Explorer**
  * Перейти в папку **\data\data\ru.iteco.fmhandroid\files\target**
  * В контекстном меню папки  выбрать пункт **Save As...**, затем указать путь к корню папки проекта для сохранения

2. Для визуализации отчета в терминале ввести команду **allure serve**
